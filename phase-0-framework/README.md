# Phase 0: Framework & Estimation

> Before designing any system, you need two things: a structured approach and the ability to do quick math.

## What You'll Learn

1. **RESHADED Framework** -- A repeatable 8-step process for any system design question
2. **Back-of-Envelope Estimation** -- Calculate QPS, storage, and bandwidth in minutes
3. **Requirements Gathering** -- Turn vague questions into concrete, scoped problems

## Why This Phase Matters

Most candidates jump straight to drawing boxes. They design a solution for a problem they haven't fully understood. Phase 0 teaches you to think before you draw.

## Modules

| # | Module | Time | What You'll Do |
|---|--------|------|---------------|
| 1 | [How to Approach](how-to-approach.md) | 2 hours | Learn the RESHADED framework step by step |
| 2 | [Estimation Cheatsheet](estimation-cheatsheet.md) | 3 hours | Memorize key numbers, practice calculations |
| 3 | [Requirements Gathering](requirements-gathering.md) | 3 hours | Practice scoping 3 example systems |

## Success Criteria

By the end of Phase 0, you should be able to:

- [ ] Structure a 45-minute system design answer using RESHADED
- [ ] Estimate QPS, storage, and bandwidth for any system in under 5 minutes
- [ ] Distinguish functional from non-functional requirements
- [ ] Scope a design problem to 3-5 core features
